# Plantilla de evidencia individual

Copiá esta página con el archivo indicado en el mapa después del único ejercicio `Para casa`. La entrada debe dejar una decisión comprobable, no volver a narrar toda la práctica.

> [Completar] Reemplazá esta línea y las ayudas de cada sección por evidencia propia antes de marcar la entrada como `Mínimo`.

## Objetivo

- **Clase y pregunta:** qué querías comprobar.
- **Criterio:** qué resultado observable respondería la pregunta.

## Configuración

Registrá dataset o fixture, split, baseline o control, parámetros relevantes, versión del entorno y presupuesto. Incluí solo lo necesario para interpretar la comparación.

## Run o traza

Anotá el Run ID, Trace ID o exportación local equivalente. Si no corresponde usar tracking, registrá el commit, comando, build o check que ubica la ejecución; no inventes identificadores.

## Resultado y comparación

Compará baseline, control o estado anterior contra el candidato. Mostrá una métrica, prueba, tabla, gráfico, log o caso de falla y explicá qué permite afirmar.

## Evidencia

Enlazá entre una y tres piezas concretas y señalá cuál debería inspeccionarse primero. No publiques `mlruns/`, modelos pesados, secretos, PII ni material A0.

## Reproducibilidad

Indicá el entorno y la forma mínima de abrir, repetir o inspeccionar el resultado. Si depende de hardware, credenciales o un servicio, dejá un fallback local, congelado o simulado.

## Decisión y límite

- **Decisión:** qué conservás y con qué criterio.
- **Alternativa descartada:** qué no elegiste y por qué.
- **Límite:** qué dato, métrica o condición impide generalizar.

## Siguiente experimento

Definí una sola variación y qué resultado confirmaría, debilitaría o cambiaría la decisión actual.

## Uso de IA

Registrá nivel A0–A4, propósito, salida aceptada o rechazada, verificación humana y datos que no compartiste con herramientas externas.

## Microdefensa

En 3–5 líneas, explicá la evidencia central, la decisión propia y qué cambiaría si varía el dato, la métrica o el requisito.
